## What this is

A hands-on workshop (instructional content in **Korean**) demonstrating an end-to-end ML
workflow on **Amazon SageMaker Unified Studio** with **MLflow** experiment tracking. It is a
sequence of Jupyter notebooks, not an installable package ‚Äî there is no build, lint, or test
suite. The notebooks are the source of truth.

## The notebooks are the only versioned code

`code/`, `processing/`, `training/`, `tmp/`, `data/`, `bank-additional/`, `mlruns/`, and model
artifacts (`*.tar.gz`, `xgboost-model`) are **all gitignored** ‚Äî they are generated at runtime.
The `.py` scripts in `code/`/`processing/`/`training/`/`pipeline_code/` are produced by `%%writefile`
cells inside the notebooks; `bank-additional/` and `data/` are downloaded by the notebooks. **Never
edit those generated files directly** ‚Äî edit the `%%writefile` cell in the owning notebook, since a
re-run overwrites them. (`pipeline_code/` is the one such directory that is **committed** rather than
gitignored, but it is still `%%writefile`-generated by notebook 5, so the same rule applies.)
To inspect a notebook as plain Python:

```bash
jupyter nbconvert --to script --stdout 2-training.ipynb
```

## Two independent tracks ‚Äî do not conflate them

- **Notebooks 0‚Üí5** ‚Äî Bank Marketing dataset, **binary classification** (predict term-deposit
  subscription, `binary:logistic`). Notebooks **0‚Üí4** run **sequentially and share state** (see
  `%store` below); **notebook 5** (`5-sagemaker-pipelines.ipynb`) automates the same workflow as a
  **SageMaker Pipelines** DAG. Notebook 5 is **not standalone** ‚Äî it restores `%store -r input_source`
  (the raw Bank Marketing CSV S3 URI, set in notebook 1), so notebooks 0 and 1 must run first. Its DAG
  trains two models in parallel (`BankTrainConservative` / `BankTrainAggressive`), evaluates both, and
  uses a `BankAUCCondition` gate to **conditionally register the better model** (`BankRegisterModel`)
  or fail the pipeline (`BankPipelineFail`).
- **Notebook 6** (`6-pipelines_preprocess_train_evaluate_batch_transform.ipynb`) ‚Äî UCI **Abalone**,
  **regression** (`reg:linear`, MSE). A **standalone** SageMaker Pipelines demo that does NOT consume
  any `%store` output of the bank track and can run alone.

The tracks have separate preprocessing/training/evaluation scripts with different schemas:
- Bank step-by-step (0‚Üí4): `processing/preprocessing.py` + `training/train.py`.
- Bank pipeline (5): `pipeline_code/preprocessing.py` + `pipeline_code/evaluation.py`
  (+ `pipeline_code/requirements.txt`).
- Abalone pipeline (6): `code/preprocessing.py` + `code/evaluation.py`.

When changing data shape, only touch the track you mean.

## Cross-notebook state via `%store`

Notebook 0 computes `bucket`, `prefix`, `role`, `region`, `mlflow_arn`, `mlflow_name`,
`experiment_name` and persists them with `%store`. Notebooks 1‚Üí5 begin with `%store -r ...` to
restore them (notebook 5 also restores `input_source`, set in notebook 1). **Notebook 0 must be
run first**, in the same Jupyter profile, or the later notebooks fail at restore.

## Environment bootstrap (notebook 0, cell 1) ‚Äî handle with care

The kernel image ships only modular/v3 SageMaker packages, so notebook 0 installs the **classic
SageMaker Python SDK v2** (`sagemaker>=2.220,<3`) needed for `FrameworkProcessor`, `Estimator`,
and `Pipelines`. Two pins matter:
- `setuptools<81` ‚Äî setuptools 81+ removes `pkg_resources`, which breaks `import mlflow` (2.13.2).
- `mlflow==2.13.2` with `sagemaker-mlflow`.

The install is wrapped in an `importlib.util.find_spec("sagemaker.sklearn")` guard that skips
reinstall + kernel restart when already present. **Preserve this guard** ‚Äî without it, "Run All"
triggers an infinite install‚Üírestart loop.

## How config is discovered (not hardcoded)

- `project = Project()` (from `sagemaker_studio`) yields the S3 root (`project.s3.root`), IAM role
  (`project.iam_role`), domain/project IDs, and Glue database.
- The **MLflow tracking ARN is looked up via the DataZone API** by paginating environments and
  partial-matching the name `"MLExperiments"` ‚Äî `project.mlflow_tracking_server_arn` is NOT used
  because the real environment name may be prefixed (e.g. `"OnCreate MLExperiments"`).

## SageMaker job conventions

- Training scripts read SageMaker env vars (`SM_MODEL_DIR`, `SM_CHANNEL_TRAIN`,
  `SM_CHANNEL_VALIDATION`, `SM_HOSTS`, etc.) and read MLflow config from `MLFLOW_TRACKING_ARN` /
  `MLFLOW_EXPERIMENT_NAME` / `REGION` passed in as environment.
- Containers install their own deps at job start (`pip install` inside the script) and treat
  MLflow as **optional** ‚Äî every MLflow call is guarded so a tracking failure never kills the job.
- CSV convention: **label is the first column, headerless**. The booster is pickled as a file
  literally named `xgboost-model`.
- Notebook 2 trains two models ‚Äî built-in XGBoost (`xgb1`/`xgb2`) and Script Mode (`train.py`) ‚Äî
  then `mlflow.register_model(...)`. Logging must succeed *before* registration, or an empty
  artifact gets registered.
